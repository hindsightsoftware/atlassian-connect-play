Preparing for production

Here are a few steps you should go through to configure you add-on and your environment before deploying it in production.

Set your add-on key (and name)

By default the module defines an add-on key for you when in development. If you haven't done so, set your add-on key (and name) in the application.conf file:

@Constants.AC_PLUGIN_KEY
your add-on key, which is used internally as a key by Atlassian Connect.
@Constants.AC_PLUGIN_NAME
your add-on name, which is used to display the name of you add-on to end users.

Set your base URL

By default the base URL is computed by the framework. When deploying to a production instance you should define the BASE_URL environment variable which will then be used as the base URL for your add-on.
